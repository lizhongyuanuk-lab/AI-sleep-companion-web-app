# Stage 3 Acceptance

## Manual verification checklist

- Fresh browser or cleared localStorage opens onboarding.
- Completing onboarding routes user into the main app.
- Refreshing after onboarding keeps user inside the main app.
- Clearing localStorage resets first launch behavior.
- Room reads generated profile and companion data.
- Talk screen can create mock user and companion messages.
- Memory screen shows onboarding-derived seed memories.
- Sleep screen can start and end a mock sleep session.
- App does not crash if localStorage is empty.
- App does not crash if localStorage contains malformed JSON.
- Existing UI styling remains materially unchanged.
